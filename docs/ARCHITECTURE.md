# SecondBrain 架构设计

## 系统概述

SecondBrain 是一个基于 Obsidian Vault 的智能知识管理系统，支持语义搜索、关键词搜索和混合检索，具备优先级分类和自动增量索引功能。

## 核心模块

### 1. 索引模块 (`src/index/`)

#### 组件图
```
┌─────────────────────────────────────────────────────────┐
│                    Index Pipeline                        │
│                                                          │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐          │
│  │  File    │───▶│  Chunker │───▶│  Embedder│          │
│  │  Watcher │    │ (800/150)│    │(fastembed)│         │
│  └──────────┘    └──────────┘    └────┬─────┘          │
│                                       │                  │
│                    ┌──────────────────┴──────────┐      │
│                    │                             │      │
│            ┌───────▼───────┐           ┌────────▼──────┐│
│            │ SemanticIndex │           │ KeywordIndex  ││
│            │  (sqlite-vec) │           │  (SQLite FTS) ││
│            │  512-dim      │           │  BM25/FTS5    ││
│            └───────┬───────┘           └────────┬──────┘│
│                    │                             │      │
│                    └───────────┬─────────────────┘      │
│                                │                        │
│                        ┌───────▼───────┐               │
│                        │ HybridRetriever│               │
│                        │  RRF + Priority│               │
│                        └────────────────┘               │
└─────────────────────────────────────────────────────────┘
```

#### 数据流
```
1. 文件变更检测 (Watchdog)
   ↓
2. 读取文件内容 (Markdown)
   ↓
3. 文本分块 (Chunker: 800 字符，150 重叠)
   ↓
4. 向量化 (Embedder: BAAI/bge-small-zh-v1.5, 512 维)
   ↓
5. 双重索引
   ├─ 语义索引 (sqlite-vec, 余弦相似度)
   └─ 关键词索引 (SQLite FTS5, BM25)
   ↓
6. 混合检索 (RRF 融合 + 优先级加权)
   ↓
7. 返回结果 (Top-K, 带元数据)
```

## 数据模型

### 索引数据结构

```sql
-- 语义索引表
CREATE TABLE vectors (
    doc_id TEXT PRIMARY KEY,      -- 文档 ID (note_001#0)
    embedding BLOB,               -- 向量 (512 维 float32)
    metadata TEXT                 -- JSON 元数据
);

-- 关键词索引 (SQLite FTS5)
CREATE VIRTUAL TABLE keywords USING fts5(
    doc_id,
    content,
    content='vectors',
    content_rowid='rowid'
);

-- 文档映射 (内存)
{
    "doc_id": {
        "chunks": 5,              -- 块数量
        "metadata": {
            "file_path": "...",
            "priority": 5,
            "tags": ["work"]
        }
    }
}
```

### 优先级模型

```yaml
priority_levels:
  - priority: 9                   # 级别 (1-9)
    tags: ["central_gov"]         # 匹配标签
    description: "国家级文件"
    retention_days: -1            # 保留天数 (-1=永久)
    weight: 2.0                   # 搜索权重
```

## 并发与性能

### 并发模型
- **单线程**: 主要逻辑运行在主线程
- **异步 I/O**: 使用 `asyncio` 处理文件 I/O
- **懒加载**: 模型和索引按需加载

### 性能优化策略
1. **分块策略**: 800 字符/块，150 字符重叠
2. **批量处理**: 批量编码和插入
3. **索引缓存**: SQLite 内存映射
4. **懒加载**: 模型和索引按需加载
5. **增量更新**: 文件变更检测 + 增量索引

### 性能指标
| 操作 | 耗时 | 备注 |
|------|------|------|
| 模型加载 | 2-5s | 首次，缓存后 0.5s |
| 向量生成 | 10-20ms/块 | BAAI/bge-small-zh-v1.5 |
| 语义搜索 | 10-50ms | 1000 文档 |
| 关键词搜索 | 5-20ms | SQLite FTS5 |
| 混合检索 | 15-70ms | RRF 融合 |
| 文档添加 | 50-100ms | 取决于大小 |

## 安全设计

### 安全策略
1. **文件访问限制**: 仅访问配置的 Vault 目录
2. **大小限制**: 最大文件大小 1MB
3. **批量限制**: 批量操作最大 20 条
4. **删除确认**: 删除前需要确认
5. **路径遍历防护**: 规范化路径，检查前缀

### 数据保护
- **软删除**: 删除文件移动到 `.trash/`
- **备份**: 定期备份索引和配置

## 故障恢复

### 容错机制
1. **索引损坏**: 自动检测 + 重建提示
2. **模型加载失败**: 降级到关键词搜索
3. **文件读取失败**: 跳过 + 日志记录
4. **数据库锁定**: 重试机制 + 超时

### 恢复策略
```bash
# 1. 检查索引状态
python3 scripts/check_index_status.py

# 2. 重建索引
python3 scripts/build_index.py --full

# 3. 恢复备份
tar -xzf backup.tar.gz -C ~/.local/share/secondbrain/
```
